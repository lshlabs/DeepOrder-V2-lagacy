# kds-web 프론트 구현 아이디어 문서

## 1. 목적

`kds-web`은 매장주 또는 매장 내부에서 사용하는 웹 KDS 화면이다.

목표는 실제 주방에서 빠르게 주문을 확인하고 처리할 수 있도록, 주문 카드 중심의 고밀도 화면으로 개선하는 것이다.

핵심 방향:

```text
- 주문 카드 중심 UI
- 불필요한 정보 축소
- 진행중 / 완료 탭 유지
- 메뉴 완료 / 미완료를 직관적으로 표시
- 직원 계정에서는 내 업무 탭으로 담당 메뉴 업무량 확인
- 운영 상태, 통계, 설정 기능 추가
- 주문 처리 속도와 가독성 강화
```

---

## 2. 기본 화면 구조

기본 탭:

```text
진행중
완료
```

`진행중` 탭에는 `NEW`, `COOKING` 주문을 표시한다.

`완료` 탭에는 `DONE` 주문을 표시한다.

주문 카드는 가로 스크롤 보드에 표시한다.

신규 주문은 기존과 반대로 **최우측에 추가**한다.

이유:

```text
왼쪽 → 오른쪽으로 시간 흐름이 자연스럽다.
오래된 주문이 왼쪽에 남아 우선 처리하기 쉽다.
신규 주문이 오른쪽에 추가되어 흐름이 직관적이다.
```

---

## 3. 역할별 화면 명칭 구분

`kds-web`에서는 로그인한 계정 역할에 따라 화면 명칭을 구분한다.

관리자 또는 매장주 계정에서는 직원 관리 페이지를 네비게이션에서 `직원`으로 표시한다.

직원 계정에서는 업무 확인용 탭을 `내 업무`로 표시한다.

정리:

```text
관리자 / 매장주 계정:
[진행중] [완료] [통계] [설정] [직원]

직원 계정:
[진행중] [완료] [내 업무]
```

`직원`은 매장주가 직원을 생성하고 관리하는 페이지다.

`내 업무`는 직원이 자신의 담당 메뉴와 현재 업무량을 확인하는 탭이다.

---

## 4. 주문 카드 롱클릭 액션

주문 카드를 길게 클릭하거나 우클릭하면 액션 팝업을 띄운다.

선택지:

```text
상세정보
제거
```

### 상세정보

주문 상세정보를 보여준다.

포함 정보:

```text
주문번호
주문 시간
메뉴
옵션
요청사항
결제금액
플랫폼
주소
연락처
```

주소와 연락처는 민감정보이므로 기본 카드에는 표시하지 않고, 상세정보 팝업에서만 보여준다.

### 제거

주문 제거 확인 모달을 띄운다.

```text
주문을 제거하시겠습니까?
[예] [아니오]
```

초기에는 실제 주문 취소가 아니라 KDS 화면에서 숨김 또는 제거 처리로 정의한다.

---

## 5. 완료 탭 주문 내역 정리

`완료` 탭에서는 새로고침 아이콘 옆에 휴지통 아이콘을 추가한다.

동작:

```text
휴지통 클릭
→ 주문완료 내역을 삭제할까요?
→ 예 / 아니오
```

목적:

```text
완료된 주문이 계속 쌓여 화면이 복잡해지는 문제 해결
영업 중 KDS 화면 정리
```

초기 구현에서는 hard delete보다 archive 또는 hidden 처리에 가깝게 가져가는 것이 안전하다.

---

## 6. 운영 상태 인디케이터

KDS 상단에 운영 상태 인디케이터를 추가한다.

상태:

```text
영업중
일시중지
영업종료
```

인디케이터를 누르면 상태 변경 팝업을 연다.

### 영업중

정상 주문 수신 상태.

### 일시중지

일정 시간 동안 주문 수신을 중지한다.

일시중지 시간 설정:

```text
기본값: 10분
- 버튼: 10분 감소
+ 버튼: 10분 증가
```

### 영업종료

당일 주문 수신 종료 상태.

---

## 7. 통계 탭

`통계` 탭을 추가한다.

초기 표시 항목:

```text
오늘 메뉴별 주문 수
오늘 총 매출
오늘 완료 주문 수
```

후속 확장:

```text
일간 / 주간 / 월간 매출
기간별 주문 수
시간대별 주문량
인기 메뉴
```

통계 탭은 실시간 주문 처리 화면과 분리해서, 매장주가 영업 흐름을 확인하는 화면으로 둔다.

---

## 8. 설정 탭

`설정` 탭을 추가한다.

설정 항목:

```text
알림 on/off
알림 사운드 설정
브레이크타임 설정
비밀번호 변경
주문 자동수락 on/off
```

### 알림 on/off

주문 도착 알림을 켜거나 끈다.

### 알림 사운드 설정

주문 도착 시 재생할 사운드를 선택한다.

### 브레이크타임 설정

특정 시간이 되면 자동으로 일시중지 상태로 전환한다.

예시:

```text
브레이크타임 시작: 15:00
중지 시간: 60분
```

### 비밀번호 변경

비밀번호 변경 성공 시 현재 세션을 로그아웃한다.

프론트 처리:

```text
accessToken 삭제
refreshToken 삭제
AuthPage로 이동
```

### 주문 자동수락 on/off

```text
ON:
주문 수신 즉시 진행중 표시

OFF:
주문 수락 버튼을 누른 뒤 진행중 표시
```

## 9. 스크롤 개선

주문 보드는 가로 스크롤을 유지한다.

단, 하단 스크롤바는 시각적으로 숨긴다.

목표:

```text
스크롤 가능
스크롤바는 보이지 않음
터치패드 / 마우스 휠 / 드래그로 탐색
PC에서는 마우스 상하 스크롤을 가로 스크롤로 치환
```

## 10. 주문 카드 배치 로직

### 짧은 카드 2행 배치

다음 조건을 만족하면 주문 카드 2개를 한 컬럼 안에 2행으로 배치할 수 있다.

```text
1. 현재 주문 카드가 짧다.
2. 다음 주문 카드도 짧다.
3. 두 주문을 2행으로 배치해도 내용이 잘리지 않는다.
```

목적:

```text
짧은 주문들이 화면을 과하게 차지하지 않게 함
한 화면에 더 많은 주문 표시
```

### 긴 카드 2열 메뉴 배치

하나의 주문 카드가 길 경우 메뉴 목록을 2열로 배치할 수 있다.

조건:

```text
메뉴 개수가 많음
카드 세로 길이가 과하게 길어짐
2열로 나누어도 가독성이 유지됨
```

## 11. 메뉴 완료 / 미완료 상호작용

메인 KDS의 `진행중` 탭에서는 주문 카드 내부의 각 메뉴를 클릭해 완료 / 미완료를 토글할 수 있다.

메뉴를 클릭하면 해당 메뉴가 완료 상태로 표시된다.

다시 클릭하면 완료 상태를 해제한다.

예시:

```text
짜장면 x1
짬뽕 x2
탕수육 x1
```

짜장면을 클릭하면:

```text
짜장면 x1  완료 상태
짬뽕 x2
탕수육 x1
```

완료 상태는 텍스트를 추가하지 않고, 배경색 변화 중심으로 표현한다.

예시:

```text
미완료 메뉴:
기본 배경

완료 메뉴:
연한 초록색 배경
```

메인 KDS는 1초 안에 주문을 파악해야 하는 공간이므로, 메뉴 옆에 추가 설명을 넣지 않는다.

메뉴 완료 상태 표현 원칙:

```text
- 메뉴명과 수량의 가독성을 해치지 않는다.
- 완료 상태는 배경색, 흐림 처리, 체크 아이콘 등 최소한의 시각 변화로 표현한다.
- 주문 카드의 전체 구조는 유지한다.
- 완료 처리는 메뉴 라인 단위로 한다.
```

수량이 여러 개인 메뉴도 초기 구현에서는 메뉴 라인 단위로 완료 / 미완료만 처리한다.

예를 들어 `짬뽕 x2`를 완료 처리하면 짬뽕 메뉴 라인 전체가 완료 상태로 표시된다.

세부 수량별 완료 처리는 초기 구현 범위에서 제외한다.

---

## 12. 내 업무 탭

직원 계정으로 로그인했을 때 상단 탭은 다음과 같이 표시한다.

```text
[진행중] [완료] [내 업무]
```

`내 업무` 탭은 직원이 완료 처리를 하는 화면이 아니라, 자신이 담당해야 하는 메뉴와 현재 남은 업무량을 확인하는 화면이다.

완료 처리는 메인 KDS의 `진행중` 탭에서만 수행한다.

---

## 13. 내 업무 탭 화면 구성

`내 업무` 탭은 크게 두 영역으로 구성한다.

```text
1. 담당 메뉴 요약 카드 영역
2. 업무 히스토리 영역
```

---

## 14. 담당 메뉴 요약 카드 영역

상단에는 직원이 등록한 담당 메뉴별 남은 수량을 카드 형태로 표시한다.

예시:

```text
[짜장면]        [짜장면 곱빼기]    [짬뽕]        [짬뽕곱빼기]
   3                 3              0              1
```

카드 구성 요소:

```text
메뉴명
현재 남은 수량
```

표시 규칙:

```text
- 카드 하나는 담당 메뉴 하나를 의미한다.
- 숫자는 현재 진행중 주문 중 아직 메인 KDS에서 완료 처리되지 않은 해당 메뉴의 총 수량이다.
- 수량이 0이어도 카드는 유지할 수 있다.
- 수량이 0인 카드는 비활성 스타일 또는 옅은 색상으로 표시할 수 있다.
```

예시:

```text
직원이 등록한 담당 메뉴:
- 짜장면
- 짜장면 곱빼기
- 짬뽕
- 짬뽕곱빼기

현재 진행중 주문 기준 남은 수량:
- 짜장면 3
- 짜장면 곱빼기 3
- 짬뽕 0
- 짬뽕곱빼기 1
```

이 경우 `내 업무` 탭 상단에는 위 메뉴들이 카드 형태로 표시된다.

---

## 15. 담당 메뉴 카드 UI 원칙

담당 메뉴 카드는 한눈에 수량을 확인하는 것이 목적이다.

따라서 카드 안에는 불필요한 설명을 넣지 않는다.

권장 표시:

```text
메뉴명
큰 숫자
```

예시:

```text
짜장면
3
```

카드 스타일:

```text

- 메뉴명은 상단 또는 중앙 상단
- 수량은 크게 표시
- 카드 간 간격은 넉넉하게 유지
- 수량이 많은 메뉴가 시각적으로 바로 들어오도록 숫자를 크게 표시
```

## 16. 업무 히스토리 영역

하단에는 현재 직원 담당 메뉴와 관련된 주문 항목을 테이블 형태로 표시한다.

예시:

```text
업무 히스토리

| 주문번호 | 메뉴       | 갯수 | 시각               | 상태   |
|---------|------------|------|--------------------|--------|
| ABCD13  | 짜장면     | 1    | 6월 17일 오후 7:43 | 진행중 |
| ABCD12  | 짬뽕곱빼기 | 1    | 6월 17일 오후 7:35 | 진행중 |
| ABCD12  | 짜장면     | 2    | 6월 17일 오후 7:35 | 진행중 |
| ABCD11  | 짬뽕       | 4    | 6월 17일 오후 7:23 | 완료   |
```

업무 히스토리의 목적은 직원이 자신의 업무가 어떤 주문에서 발생했는지 확인하는 것이다.

---

## 17. 업무 히스토리 컬럼

업무 히스토리 테이블에는 다음 컬럼을 포함한다.

```text
주문번호
메뉴
갯수
시각
상태
```

각 컬럼 의미:

```text
주문번호:
해당 메뉴가 포함된 주문 번호

메뉴:
직원이 담당 메뉴로 등록한 메뉴명

갯수:
해당 주문 안에 포함된 메뉴 수량

시각:
주문이 들어온 시각

상태:
진행중 또는 완료
```

---

## 18. 업무 히스토리 상태 표시 규칙

상태는 메인 KDS의 메뉴 완료 상태를 기준으로 표시한다.

```text
진행중:
해당 주문 메뉴가 아직 메인 KDS에서 완료 처리되지 않은 상태

완료:
해당 주문 메뉴가 메인 KDS에서 완료 처리된 상태
```

직원은 `내 업무` 탭에서 상태를 직접 변경하지 않는다.

---

## 19. 메인 KDS와 내 업무 탭 연동

`내 업무` 탭의 수량과 상태는 메인 KDS의 메뉴 완료 상태를 기준으로 자동 갱신된다.

예시:

```text
현재 내 업무 카드:
짜장면 3
짬뽕곱빼기 1
```

메인 KDS `진행중` 탭에서 주문 `ABCD13`의 짜장면 1개를 완료 처리하면:

```text
짜장면 3 → 짜장면 2
```

업무 히스토리에서는 해당 항목의 상태가 변경된다.

```text
ABCD13 / 짜장면 / 1 / 진행중
→
ABCD13 / 짜장면 / 1 / 완료
```

메인 KDS에서 메뉴 완료 상태를 해제하면 `내 업무` 탭에도 다시 진행중으로 반영된다.

---

## 20. 담당 메뉴 관리

직원은 `내 업무` 탭에서 자신이 보고 싶은 담당 메뉴를 관리할 수 있다.

필수 기능:

```text
담당 메뉴 추가
담당 메뉴 수정
담당 메뉴 삭제
```

담당 메뉴 추가 방식:

```text
[메뉴 추가] 버튼 클릭
→ 메뉴명 입력
→ 저장
```

담당 메뉴 수정 방식:

```text
기존 담당 메뉴 선택
→ 메뉴명 수정
→ 저장
```

담당 메뉴 삭제 방식:

```text
담당 메뉴 선택
→ 삭제
→ 확인 모달
```

삭제 확인 문구:

```text
이 담당 메뉴를 삭제하시겠습니까?
[예] [아니오]
```

---

## 21. 담당 메뉴 관리 UI

담당 메뉴 관리 UI는 상단 카드 영역 근처에 배치한다.

예시:

```text
내 업무
[메뉴 추가]

[짜장면 3] [짜장면 곱빼기 3] [짬뽕 0] [짬뽕곱빼기 1]
```

담당 메뉴 카드를 클릭했을 때는 완료 처리하지 않는다.

카드 클릭 동작은 다음 중 하나로 제한한다.

```text
- 해당 메뉴의 업무 히스토리 필터링
- 담당 메뉴 수정 메뉴 열기
```

완료 처리는 절대 `내 업무` 탭에서 하지 않는다.

---

## 22. 빈 상태

등록된 담당 메뉴가 없을 때는 다음 문구를 표시한다.

```text
아직 등록된 담당 메뉴가 없습니다.
메뉴를 추가하면 내 업무량을 확인할 수 있습니다.
```

현재 진행중 주문에 담당 메뉴가 없을 때는 다음 문구를 표시한다.

```text
현재 처리할 담당 메뉴가 없습니다.
```

---

## 23. 내 업무 탭 UI 원칙

```text
- 내 업무 탭은 완료 처리 화면이 아니다.
- 내 업무 탭은 현재 담당 메뉴의 남은 수량을 확인하는 화면이다.
- 상단은 카드 형태로 메뉴별 수량을 크게 표시한다.
- 하단은 테이블 형태로 업무 히스토리를 표시한다.
- 완료 상태는 메인 KDS의 메뉴 완료 상태를 기준으로 자동 반영한다.
- 직원이 내 업무 탭에서 완료 버튼을 누르는 흐름은 만들지 않는다.
```

## 24. 신규 주문 강조 효과

주문이 새로 도착하면 해당 카드에 일시적인 outer glow 또는 breathe 효과를 적용한다.

```text
신규 주문 도착
→ 카드 외곽 glow
→ 일정 시간 후 사라짐
```

주방에서 새 주문을 즉시 인지하기 위한 시각 피드백이다.

---

## 25. 새로고침 버튼 개선

새로고침 버튼은 실제 주문 목록을 갱신해야 한다.

동작:

```text
새로고침 클릭
→ 아이콘 한 바퀴 회전
→ 주문 영역 갱신
→ 주문 카드 fade-out / fade-in
```

---

## 26. Auth 페이지 전환 애니메이션

로그인 / 매장가입 탭 전환 시 fade 효과를 적용한다.

```text
로그인 → 매장가입
매장가입 → 로그인
```

효과:

```text
fade-out
fade-in
```

회원가입 플로우의 완성도를 높인다.

---

## 27. 향후 직원 관리 페이지

`kds-web`에는 매장주 또는 관리자 계정용 직원 관리 페이지를 추가한다.

관리자 계정에서 직원 관리 페이지는 네비게이션에 `직원`으로 표시한다.

직원 관리 페이지 기능:

```text
직원 목록
직원 추가
직원 PIN 발급
직원 비활성화
직원 상태 확인
```

직원 관리 페이지는 매장주가 직원을 생성하고 관리하는 화면이다.

직원 계정에서 보이는 `내 업무` 탭과는 역할이 다르다.

## 28. 완료 기준

```text
주문 카드가 화면의 중심이 된다.
민감정보는 기본 카드에 노출되지 않는다.
운영 상태 변경이 가능하다.
완료 탭 내역을 정리할 수 있다.
통계 / 설정 탭으로 매장 운영 기능이 확장된다.
스크롤바는 숨기되 가로 스크롤은 유지된다.
신규 주문과 새로고침에 명확한 피드백이 있다.
메인 KDS 진행중 탭에서 메뉴 완료 / 미완료를 토글할 수 있다.
메뉴 완료 상태는 배경색 등 시각적 변화로 표시된다.
직원 계정에서는 [진행중] [완료] [내 업무] 탭을 제공한다.
내 업무 탭은 담당 메뉴별 남은 수량을 카드 형태로 표시한다.
내 업무 탭은 업무 히스토리를 테이블 형태로 표시한다.
내 업무 탭에서는 완료 처리를 하지 않는다.
메인 KDS에서 메뉴가 완료되면 내 업무 탭의 수량과 상태가 자동 갱신된다.
관리자 계정의 직원 관리 페이지는 [직원]으로 표시한다.
기존 Auth / store context / KDS API 로직은 유지된다.
```
