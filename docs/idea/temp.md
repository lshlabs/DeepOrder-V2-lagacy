# 1. kds-web 프론트 구현 아이디어 문서

파일명 제안: `kds-web-frontend-ideas.md`

## 1. 목적

`kds-web`은 매장주 또는 매장 내부에서 사용하는 웹 KDS 화면이다.

목표는 실제 주방에서 빠르게 주문을 확인하고 처리할 수 있도록, 주문 카드 중심의 고밀도 화면으로 개선하는 것이다.

핵심 방향:

```text
- 주문 카드 중심 UI
- 불필요한 정보 축소
- 진행중 / 완료 탭 유지
- 운영 상태, 통계, 설정 기능 추가
- 주문 처리 속도와 가독성 강화
```

---

## 2. 기본 화면 구조

기본 탭:

```text
진행중
완료
```

`진행중` 탭에는 `NEW`, `COOKING` 주문을 표시한다.

`완료` 탭에는 `DONE` 주문을 표시한다.

주문 카드는 가로 스크롤 보드에 표시한다.

신규 주문은 기존과 반대로 **최우측에 추가**한다.

이유:

```text
왼쪽 → 오른쪽으로 시간 흐름이 자연스럽다.
오래된 주문이 왼쪽에 남아 우선 처리하기 쉽다.
신규 주문이 오른쪽에 추가되어 흐름이 직관적이다.
```

---

## 3. 주문 카드 롱클릭 액션

주문 카드를 길게 클릭하면 액션 팝업을 띄운다.

선택지:

```text
상세정보
제거
```

### 상세정보

주문 상세정보를 보여준다.

포함 정보:

```text
주문번호
주문 시간
메뉴
옵션
요청사항
결제금액
플랫폼
주소
연락처
```

주소와 연락처는 민감정보이므로 기본 카드에는 표시하지 않고, 상세정보 팝업에서만 보여준다.

### 제거

주문 제거 확인 모달을 띄운다.

```text
주문을 제거하시겠습니까?
[예] [아니오]
```

초기에는 실제 주문 취소가 아니라 KDS 화면에서 숨김 또는 제거 처리로 정의한다.

---

## 4. 완료 탭 주문 내역 정리

`완료` 탭에서는 새로고침 아이콘 옆에 휴지통 아이콘을 추가한다.

동작:

```text
휴지통 클릭
→ 주문완료 내역을 삭제할까요?
→ 예 / 아니오
```

목적:

```text
완료된 주문이 계속 쌓여 화면이 복잡해지는 문제 해결
영업 중 KDS 화면 정리
```

초기 구현에서는 hard delete보다 archive 또는 hidden 처리에 가깝게 가져가는 것이 안전하다.

---

## 5. 운영 상태 인디케이터

KDS 상단에 운영 상태 인디케이터를 추가한다.

상태:

```text
영업중
일시중지
영업종료
```

인디케이터를 누르면 상태 변경 팝업을 연다.

### 영업중

정상 주문 수신 상태.

### 일시중지

일정 시간 동안 주문 수신을 중지한다.

일시중지 시간 설정:

```text
기본값: 10분
- 버튼: 10분 감소
+ 버튼: 10분 증가
```

### 영업종료

당일 주문 수신 종료 상태.

---

## 6. 통계 탭

`통계` 탭을 추가한다.

초기 표시 항목:

```text
오늘 메뉴별 주문 수
오늘 총 매출
오늘 완료 주문 수
```

후속 확장:

```text
일간 / 주간 / 월간 매출
기간별 주문 수
시간대별 주문량
인기 메뉴
직원별 작업 완료 수
```

통계 탭은 실시간 주문 처리 화면과 분리해서, 매장주가 영업 흐름을 확인하는 화면으로 둔다.

---

## 7. 설정 탭

`설정` 탭을 추가한다.

설정 항목:

```text
알림 on/off
알림 사운드 설정
브레이크타임 설정
비밀번호 변경
주문 자동수락 on/off
```

### 알림 on/off

주문 도착 알림을 켜거나 끈다.

### 알림 사운드 설정

주문 도착 시 재생할 사운드를 선택한다.

### 브레이크타임 설정

특정 시간이 되면 자동으로 일시중지 상태로 전환한다.

예시:

```text
브레이크타임 시작: 15:00
중지 시간: 60분
```

### 비밀번호 변경

비밀번호 변경 성공 시 현재 세션을 로그아웃한다.

프론트 처리:

```text
accessToken 삭제
refreshToken 삭제
AuthPage로 이동
```

### 주문 자동수락 on/off

```text
ON:
주문 수신 즉시 진행중 표시

OFF:
주문 수락 버튼을 누른 뒤 진행중 표시
```

## 8. 스크롤 개선

주문 보드는 가로 스크롤을 유지한다.

단, 하단 스크롤바는 시각적으로 숨긴다.

목표:

```text
스크롤 가능
스크롤바는 보이지 않음
터치패드 / 마우스 휠 / 드래그로 탐색
PC에서는 마우스 상하 스크롤을 가로 스크롤로 치환
```

---

## 9. 주문 카드 배치 로직

### 짧은 카드 2행 배치

다음 조건을 만족하면 주문 카드 2개를 한 컬럼 안에 2행으로 배치할 수 있다.

```text
1. 현재 주문 카드가 짧다.
2. 다음 주문 카드도 짧다.
3. 두 주문을 2행으로 배치해도 내용이 잘리지 않는다.
```

목적:

```text
짧은 주문들이 화면을 과하게 차지하지 않게 함
한 화면에 더 많은 주문 표시
```

### 긴 카드 2열 메뉴 배치

하나의 주문 카드가 길 경우 메뉴 목록을 2열로 배치할 수 있다.

조건:

```text
메뉴 개수가 많음
카드 세로 길이가 과하게 길어짐
2열로 나누어도 가독성이 유지됨
```

---

## 10. 메뉴별 완료 상호작용

주문 카드 내부의 메뉴 하나하나를 클릭 가능하게 한다.

동작:

```text
메뉴 클릭
→ 색상 변경
→ 완료 표시
```

이 기능은 향후 직원 개인 음식 탭의 완료 숫자와 연결될 수 있다.

---

## 11. 신규 주문 강조 효과

주문이 새로 도착하면 해당 카드에 일시적인 outer glow 효과를 적용한다.

```text
신규 주문 도착
→ 카드 외곽 glow
→ 일정 시간 후 사라짐
```

주방에서 새 주문을 즉시 인지하기 위한 시각 피드백이다.

---

## 12. 새로고침 버튼 개선

새로고침 버튼은 실제 주문 목록을 갱신해야 한다.

동작:

```text
새로고침 클릭
→ 아이콘 한 바퀴 회전
→ 주문 영역 갱신
→ 주문 카드 fade-out / fade-in
```

---

## 13. Auth 페이지 전환 애니메이션

로그인 / 매장가입 탭 전환 시 fade 효과를 적용한다.

```text
로그인 → 매장가입
매장가입 → 로그인
```

효과:

```text
fade-out
fade-in
```

회원가입 플로우의 완성도를 높인다.

---

## 14. 향후 직원 관리 탭

`kds-web`에는 향후 직원 관리 탭을 추가한다.

직원 관리 탭 기능:

```text
직원 목록
직원 추가
직원 PIN 발급
직원 비활성화
직원 상태 확인
```

직원 개인 음식 탭 자체는 직원이 `kds-app`에서 직접 관리하는 방향이지만, 매장주는 `kds-web`에서 직원을 생성하고 관리한다.

---

## 15. 완료 기준

```text
주문 카드가 화면의 중심이 된다.
민감정보는 기본 카드에 노출되지 않는다.
운영 상태 변경이 가능하다.
완료 탭 내역을 정리할 수 있다.
통계 / 설정 탭으로 매장 운영 기능이 확장된다.
스크롤바는 숨기되 가로 스크롤은 유지된다.
신규 주문과 새로고침에 명확한 피드백이 있다.
기존 Auth / store context / KDS API 로직은 유지된다.
```
